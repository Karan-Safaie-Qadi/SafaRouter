v1.3.0 — New Features

SafaRouter v1.3.0 introduces professional-grade error handling, access control, and more.

  • ErrorManager — handles all HTTP status codes (400–511) with per-status enable/disable, custom error page loading, LRU caching, error grouping (client/server), stack trace control, pluggable logger, and redirect rules
  • AccessController — blocked routes (403) and ignored routes (silent 404) with glob-style patterns (* single-level, ** recursive)
  • Maintenance Mode — toggleable via router.setMaintenance(true/false) with allowed path bypasses
  • Route data loaders — async functions that run before page render, passing data to the page component
  • Route guards — per-route guard functions that can redirect unauthenticated users
  • Per-route transitions — each route can define custom transition effects via meta.transition
  • New events: EVENTS.ACCESS_DENIED, EVENTS.MAINTENANCE
  • New methods: blockRoute(), unblockRoute(), ignoreRoute(), unignoreRoute(), isMaintenance(), setMaintenance(), retry()

Route Tree Structure

Routes are defined as a nested object. Each key is a URL segment. The tree mirrors the Next.js App Router mental model.

{
  '/': { page, layout },
  '/blog': {
    page,
    children: {
      '[slug]': { page }
    }
  }
}

Dynamic Segments

[param] matches a single path segment and exposes it via params.param.

[...param] matches one or more segments as an array.

[[...param]] matches zero or more segments.

Route Groups

(groupName) organises routes without affecting the URL.

'(main)': {
  children: {
    'profile': {
      children: {
        '[id]': { page }
      }
    }
  }
}

This makes /profile/me available without /main/profile/me.

Middleware

Middleware intercepts every navigation. Use it for auth, redirects, logging, or analytics.

router.use(async (ctx, next) => {
  console.log('Navigating to', ctx.path)
  return next()
})

Hash Mode

Set useHash: true in config for hash-based routing. Useful when you can't configure your server for SPA fallback.

const router = new SafaRouter({
  useHash: true,
  target: '#app',
  routes: { '/': { page: home } }
})

ErrorManager API

// All HTTP status codes handled
router.errorManager.setStatusEnabled(404, false)  // disable 404
router.errorManager.setGroupEnabled('server-error', false)  // disable 5xx

// Custom error pages loaded from html-pages/errors/{code}.html
router.errorManager.resolvePage(404, 'html-pages/errors')

// Error logging with custom handler
router.errorManager.setLogHandler(entry => {
  console.log('[' + entry.statusCode + '] ' + entry.path, entry.error)
})

AccessController API

// Blocked routes (403)
router.blockRoute('/admin')
router.blockRoute('/private/**')

// Ignored routes (silent 404)
router.ignoreRoute('/hidden')
router.ignoreRoute('/deprecated/*')

// Check access
router.accessController.isAccessible('/admin')   // false
router.accessController.isAccessible('/about')    // true

Maintenance Mode

// Enable maintenance
router.setMaintenance(true)

// Check status
router.isMaintenance()  // true

// Routes that bypass maintenance
config: {
  maintenanceMode: {
    allowedPaths: ['/login', '/assets/**']
  }
}

Route Loaders & Guards

'/dashboard': {
  page: dashboardPage,
  guard: ({ params, query, router }) => {
    const authed = checkAuth()
    return authed || '/login'
  },
  loader: async ({ params, query, router }) => {
    const data = await fetch('/api/data')
    return data
  },
}

Event System

SafaRouter emits events throughout the navigation lifecycle. Subscribe with router.on(event, fn).

router.on('routechange', ({ pathname, params }) => {
  console.log('Navigated to', pathname, params)
})

router.on('error', ({ path, error }) => {
  console.error('Error at', path, error)
})

router.on('accessdenied', ({ path, reason }) => {
  console.warn('Access denied to', path, reason)
})

router.on('maintenance', ({ path }) => {
  console.info('Maintenance mode, blocked:', path)
})
