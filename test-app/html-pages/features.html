Features

A comprehensive overview of everything SafaRouter offers out of the box — zero dependencies, fully tested.

🗂️ Routing

Nested Routes

Define routes as a deep object tree mirroring your URL structure. Each segment maps to a file or inline component.

Dynamic Routes

Use [param], [...catchAll], and [[...optional]] segments to match variable URL patterns.

Auto-Resolve

Pages are automatically resolved from pageDir by path. No manual imports for most routes.

Route Groups

Organise routes with (group) syntax without affecting the URL path.

Hash Mode

Support for hash-based routing (useHash: true) for environments without History API fallback.

📐 Layouts

Nested Layouts

Per-route layouts that wrap child routes. Layouts can be nested arbitrarily deep, just like Next.js App Router.

Global Layout

Define a top-level layout via the layout config option that wraps every page.

🧩 Components

Smart Components

Functions that return HTML strings, re-rendered on every navigation. Receive route context (path, params, query). Perfect for dynamic headers, footers, sidebars.

hideComponents

Per-route meta option to hide specific smart components (e.g. footer on certain pages). Set to true to hide all.

🔒 Access Control

Block Routes

Define blocked route patterns that return 403 Forbidden. Supports glob patterns (* single-level, ** recursive).

Allow Routes

Define allowed route patterns that explicitly grant access when in allowlist mode.

Ignore Routes

Define ignored route patterns that return silent 404. Makes routes appear non-existent.

Blocklist / Allowlist Modes

Toggle between blocklist (default — allow all except blocked/ignored) and allowlist (deny all except explicitly allowed). Switch at runtime via router.accessController.setMode().

⚠️ Error Handling

Error Pages

Custom HTML error pages for any HTTP status code (400–511). Loaded from {code}.html, with group fallback (4xx.html, 5xx.html) and built-in fallback.

notFound Pages

Built-in or custom 404 handling for unmatched routes.

Custom Errors

Per-status enable/disable via setStatusEnabled() and per-group control (client-error, server-error).

Error Logging

Pluggable error logger with timestamp, path, status code, and error details. Custom log handler via setLogHandler().

🛡️ Middleware & Guards

Middleware

Global middleware pipeline via router.use(fn). Intercepts every navigation for auth, redirects, logging, analytics, or context augmentation. Async support with next() chaining.

Route Guards

Per-route guard functions that run before render. Return true to allow or a redirect URL to block. Ideal for authentication checks.

📡 Real-time

SSE (Server-Sent Events)

Real-time page updates via SSE. The dev server pushes changes and the router re-fetches only the current page content.

Polling

Fallback polling mode for environments where SSE is unavailable.

WebSocket

WebSocket-based real-time updates for bidirectional communication with the dev server.

🔧 Dev Server

Live Reload

The built-in SafaDevServer watches HTML files and hot-reloads the current page via SSE — no full browser refresh.

SPA Fallback

The dev server serves index.html for all unmatched routes, enabling deep-linking in single-page applications.

🎨 Transitions

Per-route page transitions with configurable CSS classes, duration, enter/exit animations. Define via meta.transition in route config. Works with any CSS animation or transition framework.

🖥️ SSR

SafaRouter supports server-side rendering by providing the router state via router.getState() and rendering HTML on the server. The client then hydrates and takes over navigation. Works with any SSR framework.

📘 TypeScript Support

Full TypeScript definitions included (safa-router.d.ts). Covers all config options, route definitions, event payloads, error types, and method signatures for type-safe development.

⚙️ Configuration & Events

basePath

Deploy under any subdirectory with the basePath option. All routes are prefixed automatically.

Event System

Rich event lifecycle: routechange, error, accessdenied, maintenance, beforenavigation, afterrender, and more.

LRU Cache

Loaded pages are cached with configurable LRU (default 50 entries) for instant repeat navigation.

Navigation Timeout

Configurable navigation timeout with automatic error reporting on timeout.

🏗️ Maintenance Mode

Toggleable maintenance mode that returns 503 Service Unavailable for all routes except explicitly allowed bypass paths. Control at runtime via router.setMaintenance().

📂 Data Loaders

Async per-route data loaders that run before page render. Receive params, query, and router context. Returned data is available to the page component.
