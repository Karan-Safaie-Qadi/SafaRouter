Error Manager Demo

The ErrorManager handles all HTTP status codes (400–511) with custom error pages, per-status control, and error logging.

Live Error Pages

These navigate to routes that trigger real HTTP errors handled by the ErrorManager. Custom HTML pages are loaded from html-pages/errors/{code}.html.

Simulated Error Codes

These navigate to a dynamic route that simulates each error code:

Custom Error Page Files

html-pages/errors/
  403.html   ← shown when AccessController blocks a route
  404.html   ← shown when route is not found
  418.html   ← available for custom use
  500.html   ← available for custom use
  503.html   ← shown in maintenance mode

The ErrorManager first tries {code}.html, then group files (4xx.html, 5xx.html), and falls back to built-in pages.
