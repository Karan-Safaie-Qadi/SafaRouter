FAQ

Frequently asked questions about SafaRouter.

Does SafaRouter work with React / Vue / Svelte?

Yes. SafaRouter is framework-agnostic. See the React demo or Vue demo pages for integration examples.

Do I need a bundler?

No. SafaRouter runs directly in the browser via ESM imports. A bundler is optional.

Can I use it with TypeScript?

Yes. Type definitions are included (safa-router.d.ts).

How does the real-time hot reload work?

The dev server watches HTML files and sends SSE events to the browser. The router re-fetches only the current page content — no full browser refresh.

Is SafaRouter production-ready?

SafaRouter is used in development and production. It has 180+ tests and zero runtime dependencies. As with any tool, evaluate it against your project's requirements.

How is this different from React Router or Vue Router?

SafaRouter is framework-agnostic and supports nested layouts, dynamic routes, middleware, error management, access control, and real-time updates — all without requiring React, Vue, or a bundler.

Does it support hash routing?

Yes. Set useHash: true in the config.

Does SafaRouter support SSR or SSG?

SafaRouter is a client-side router. For SSR/SSG you would pair it with a static site generator or server-side framework that pre-renders HTML pages, then let SafaRouter handle client-side navigation.

How do I configure base path for subdirectory deployment?

Use the basePath option in the router config and the SafaDevServer for local development. All routes will be prefixed automatically.

const router = new SafaRouter({
  basePath: '/my-app',
  target: '#app',
  pageDir: 'html-pages',
})
