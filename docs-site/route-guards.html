Route Guards

Route guards are per-route protection functions that run before the page loads. They can allow, redirect, or block access to routes.

Basic Guard

routes: {
  '/dashboard': {
    guard: ({ params, query, router }) => {
      const token = localStorage.getItem('auth_token')
      if (!token) return '/login'  // redirect
      return true                  // allow access
    },
  },
}

Guard Return Values

ReturnBehavior
trueAllow navigation — continue loading the route
falseRedirect to /
string (URL)Redirect to the specified path
guard: ({ params }) => {
  // Allow
  if (params.id) return true

  // Redirect to home
  return false

  // Redirect to custom path
  return '/login'
}

Practical Examples

Authentication Guard

routes: {
  '/dashboard': {
    guard: ({ router }) => {
      const authed = localStorage.getItem('auth') === 'true'
      return authed || '/login'
    },
  },

  '/admin': {
    guard: ({ router }) => {
      const role = localStorage.getItem('user_role')
      return role === 'admin' || '/403'
    },
  },
}

Param Validation

routes: {
  '/post/[id]': {
    guard: ({ params }) => {
      const id = parseInt(params.id, 10)
      if (isNaN(id) || id < 1) return '/posts'
      return true
    },
  },
}

Feature Flag Guard

routes: {
  '/beta': {
    guard: () => {
      return featureFlags.betaEnabled ? true : '/'
    },
  },
}

Guards + Access Controller

SafaRouter also provides a global AccessController for pattern-based route blocking:

const router = new SafaRouter({
  access: {
    blocked: ['/admin', '/private/**'],
    ignored: ['/hidden', '/deprecated/*'],
    allowed: ['/login'],            // allowlist
    mode: 'blocklist',              // or 'allowlist'
  },
})

Runtime control:

router.blockRoute('/api/temp')
router.unblockRoute('/api/temp')
router.ignoreRoute('/old-page')
router.unignoreRoute('/old-page')

router.accessController.isAccessible('/admin')   // false
router.accessController.getMode()                // 'blocklist'

// Switch to allowlist mode
router.accessController.setMode('allowlist')
router.accessController.allow('/sandbox')

Access denied event:

router.onAccessDenied(({ path, reason }) => {
  console.warn(`Access denied: ${path} — ${reason}`)
})

Guards vs Middleware

Route GuardsMiddleware
ScopePer-routeGlobal (all routes)
Context{ params, query, router }{ path, method, query, redirect, cancelled }
Returntrue, false, or redirect pathMutate context + call next()
Use caseRoute-specific authorizationGlobal auth, logging, redirects
OrderRuns after middlewareRuns before guards
💡

Use guards for route-specific logic and middleware for global concerns. Guards have access to params, making them ideal for parameter-based authorization.
