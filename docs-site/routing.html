Routing

SafaRouter gives you two approaches to routing: HTML-first auto-routing and explicit route configuration.

HTML-First Auto-Routing

When you set pageDir, each .html file in that directory becomes a route automatically. The URL path maps directly to the file structure:

html-pages/
├── index.html          →  /
├── about.html          →  /about
├── blog/
│   ├── index.html      →  /blog
│   └── [slug].html     →  /blog/:slug
import { SafaRouter } from 'safa-router'

new SafaRouter({
  target: '#app',
  pageDir: 'html-pages',
}).start()

Explicit Route Configuration

For advanced control, define routes as a nested object:

const router = new SafaRouter({
  target: '#app',
  routes: {
    '/': {
      page: () => '<h1>Home</h1>',
    },
    '/about': {
      page: () => '<h1>About</h1>',
      meta: { title: 'About Us' },
    },
    '/blog': {
      layout: ({ children }) => `
        <div class="blog-layout">${children}</div>
      `,
      children: {
        '[slug]': {
          loader: async ({ params }) => {
            const res = await fetch(`/api/posts/${params.slug}`)
            return res.json()
          },
        },
      },
    },
  },
})

Routes defined without a page field auto-resolve from pageDir if it is set.

Navigation Methods

// Push — add a new history entry
router.push('/about')
router.push('/blog/my-post', { scroll: true })

// Replace — replace the current history entry
router.replace('/home')

// Back / Forward
router.back()
router.forward()

// Reload current route
router.reload()

// Named route navigation
router.pushRoute('blogPost', { slug: 'hello-world' }, { page: 2 })

History Mode vs Hash Mode

History Mode (default)

Uses the History API for clean URLs like /about. Requires server-side SPA fallback (all routes serve index.html):

const router = new SafaRouter({
  target: '#app',
  // useHash: false (default)
})

Hash Mode

Uses URL hash for routing — no server configuration needed. URLs look like /#/about:

const router = new SafaRouter({
  target: '#app',
  useHash: true,
})

Base Path

When your app is served from a subdirectory, set basePath:

const router = new SafaRouter({
  target: '#app',
  basePath: '/my-app',
})

All routes will be relative to /my-app/.

Linking

Use the data-safa-link attribute on anchor tags for client-side navigation:

<a href="/about" data-safa-link>About</a>

Or create a Link instance programmatically:

const link = router.createLink({
  href: '/about',
  activeClass: 'active',
})

Prefetching

Prefetch routes for faster navigation:

// Manual prefetch
router.prefetch('/about')

// Auto-prefetch when link is visible
const router = new SafaRouter({
  prefetchStrategy: 'visible', // 'none' | 'visible' | 'all'
})

Route Caching

Loaded pages are cached in an LRU cache to avoid redundant fetches:

const router = new SafaRouter({
  cacheRoutes: true,    // default
  maxCacheSize: 50,     // max cached pages
})

router.clearCache()     // clear the cache

Title Template

Automatically set document titles:

const router = new SafaRouter({
  titleTemplate: '%s — My App',
})

// Route meta title takes precedence:
// '/about': { meta: { title: 'About' } }
// → document.title: "About — My App"
