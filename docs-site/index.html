SafaRouter

A standalone frontend router inspired by Next.js App Router — works with any framework or vanilla JS

Get Started View on GitHub

Why SafaRouter?

Pure JavaScript, zero dependencies, under 5 KB gzipped. Brings the App Router development pattern to any frontend project.

⚡

HTML-First Routing

Write plain .html files — each file becomes a route automatically. No build step needed.

📁

Nested Layouts

Compose layouts from parent to child. Global layouts, per-route layouts, and HTML file layouts with {children} placeholders.

🔀

Dynamic Routes

[param], [...catchAll], [[...optional]], and route groups (group).

🛡️

Middleware & Guards

Intercept every navigation with middleware. Add per-route guard functions with redirect support.

📦

Lazy Loading

Dynamic imports and code splitting. Route-level loading states and SWR data caching.

🔗

Framework Agnostic

Works with React, Vue, Svelte, or vanilla JavaScript. No framework lock-in.

🔄

Real-Time Hot Reload

SSE-based live reload — edit HTML files, page updates without browser refresh.

🎯

Type-Safe Routes

Full TypeScript inference with defineRoute(), defineRoutes(), and SafeRouteParams<T>.

🧩

Modular Architecture

Tree-shakeable optional features. Core router under 3 KB gzipped. Vite plugin auto-detects needed features.

Quick Start

Install

npm install safa-router

Create HTML pages

Create a folder of .html files — each file is a route.

Initialize

Create a router instance, point it at your pages, and call .start().

import { SafaRouter } from 'safa-router'

new SafaRouter({
  target: '#app',
  pageDir: 'html-pages',
}).start()

That's it. Your pages are now routing. Learn more →
