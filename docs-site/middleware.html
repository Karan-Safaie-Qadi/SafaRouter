Middleware

Middleware intercepts every navigation before it happens. Use it for global auth checks, analytics, logging, redirects, or cancelling navigation.

Basic Usage

router.use(async (ctx, next) => {
  console.log(`Navigating to: ${ctx.path}`)

  // Redirect if not authenticated
  if (ctx.path.startsWith('/admin') && !isAdmin()) {
    ctx.redirect = '/login'
    return
  }

  // Cancel the navigation
  if (!featureEnabled) {
    ctx.cancelled = true
    return
  }

  return next()  // continue to next middleware / navigation
})

Middleware Context

The context object passed to each middleware function:

PropertyTypeDescription
pathstringTarget URL path being navigated to
methodstring'push' or 'replace'
queryobjectURL query parameters
redirectstring | nullSet to a path to redirect navigation
cancelledbooleanSet to true to cancel navigation

Practical Examples

Authentication Guard

router.use((ctx, next) => {
  const publicPaths = ['/', '/login', '/register']
  const token = localStorage.getItem('auth_token')

  if (!token && !publicPaths.includes(ctx.path)) {
    ctx.redirect = '/login'
    return
  }

  return next()
})

Analytics & Logging

router.use((ctx, next) => {
  const start = performance.now()

  // Run after navigation completes
  router.afterEach(() => {
    const duration = performance.now() - start
    sendAnalytics({ path: ctx.path, duration })
  })

  return next()
})

Role-Based Access

router.use((ctx, next) => {
  const user = getUser()

  if (ctx.path.startsWith('/admin') && user.role !== 'admin') {
    ctx.redirect = '/'
    return
  }

  if (ctx.path.startsWith('/moderator') && user.role !== 'moderator') {
    ctx.redirect = '/403'
    return
  }

  return next()
})

Feature Flag Check

router.use((ctx, next) => {
  if (ctx.path === '/new-feature' && !featureFlags.newFeature) {
    ctx.cancelled = true
    return
  }
  return next()
})

Middleware Chain API

SafaRouter's middleware system supports named middleware with ordering control:

// Named middleware with priority (lower runs first)
router.useNamed('auth', async (ctx, next) => {
  if (!isAuthenticated()) {
    ctx.redirect = '/login'
    return
  }
  return next()
}, 10)

router.useNamed('analytics', async (ctx, next) => {
  trackPageView(ctx.path)
  return next()
}, 20)

// Insert before/after specific middleware
router.insertMiddlewareBefore('analytics', (ctx, next) => {
  console.log('Before analytics')
  return next()
})

router.insertMiddlewareAfter('auth', (ctx, next) => {
  console.log('After auth')
  return next()
})

// Remove middleware by name
router.middleware('analytics')

Lifecycle Events

Middleware complements the event system. Use events to react to navigation outcomes:

// Before navigation starts
router.onBeforeNavigate(({ path, method }) => {
  console.log(`Starting navigation to ${path}`)
})

// After successful navigation
router.afterEach(({ pathname }) => {
  console.log(`Successfully navigated to ${pathname}`)
})

// On error
router.onError(({ path, error, statusCode }) => {
  console.error(`Error ${statusCode} at ${path}:`, error)
})
💡

Middleware is global. It runs on every navigation, including push(), replace(), back(), and forward(). For per-route protection, use Route Guards.
