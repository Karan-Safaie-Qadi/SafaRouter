Nested Routes & Layouts

Layouts wrap page content and compose from parent to child. This allows for deeply nested UI structures without repeating markup.

Global Layout

A layout applied to every route:

const router = new SafaRouter({
  layout: ({ children, params, router }) => `
    <header>
      <nav>
        <a href="/" data-safa-link>Home</a>
        <a href="/blog" data-safa-link>Blog</a>
      </nav>
    </header>
    <main>${children}</main>
    <footer>Powered by SafaRouter</footer>
  `,
})

HTML File as Layout

Use an HTML file with a {children} placeholder:

<!-- html-pages/_layout.html -->
<header>
  <nav>
    <a href="/" data-safa-link>Home</a>
    <a href="/blog" data-safa-link>Blog</a>
  </nav>
</header>
<main>{children}</main>
const router = new SafaRouter({
  layout: 'html-pages/_layout.html',
})

Per-Route Layouts

Each route can define its own layout. Layouts nest automatically:

routes: {
  '/dashboard': {
    layout: ({ children, params, router }) => `
      <div class="dashboard">
        <aside class="sidebar">
          <a href="/dashboard/settings" data-safa-link>Settings</a>
        </aside>
        <div class="content">${children}</div>
      </div>
    `,
    children: {
      'settings': {
        page: () => '<h1>Settings</h1>',
      },
      'profile': {
        page: () => '<h1>Profile</h1>',
      },
    },
  },
}

Layout cascade: globalLayout → dashboardLayout → page

Nested Route Hierarchy

Routes can be deeply nested, with each level adding its own layout:

routes: {
  '/': { page: () => '<h1>Home</h1>' },

  '/blog': {
    layout: blogLayout,       // wraps all blog routes
    children: {
      '[slug]': {
        layout: postLayout,   // wraps individual posts
        page: postPage,
        children: {
          'comments': {
            page: commentsPage,
          },
        },
      },
    },
  },
}

// /blog/my-post → blogLayout > postLayout > postPage
// /blog/my-post/comments → blogLayout > postLayout > commentsPage

Route Groups (group)

Route groups let you organize routes without affecting the URL structure — perfect for shared layouts:

routes: {
  '(auth)': {
    layout: authLayout,
    children: {
      'login': { page: loginPage },
      'register': { page: registerPage },
    },
  },
  '(main)': {
    layout: mainLayout,
    children: {
      'dashboard': { page: dashboardPage },
      'profile': { page: profilePage },
    },
  },
}

// URLs: /login, /register, /dashboard, /profile
// No "/auth/login" or "/main/dashboard" in the URL

Layout Context

Layout functions receive a context object:

layout = ({ children, params, query, router, data }) => {
  // children — the rendered child content
  // params — dynamic route parameters
  // query — URL query parameters
  // router — the SafaRouter instance
  // data — resolved data from the route's loader

  return `<div>${children}</div>`
}

Smart Components

In addition to layouts, SafaRouter supports smart components — persistent UI elements like headers and footers that render on every page:

// components/Header.js
export default function header({ path }) {
  return `
    <header>
      <nav>
        <a href="/" data-safa-link class="${path === '/' ? 'active' : ''}">Home</a>
        <a href="/about" data-safa-link class="${path === '/about' ? 'active' : ''}">About</a>
      </nav>
    </header>
  `
}

// Router config
const router = new SafaRouter({
  components: {
    header: headerComponent,
    footer: footerComponent,
  },
})
<div data-safa-component="header"></div>
<main id="app"></main>
<div data-safa-component="footer"></div>

Per-Route Component Visibility

Hide smart components on specific routes using meta.hideComponents:

routes: {
  '/sandbox': {
    meta: { hideComponents: ['footer'] },
  },
  '/plain': {
    meta: { hideComponents: true },  // hide all components
  },
}
