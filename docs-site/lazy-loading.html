Lazy Loading

SafaRouter supports lazy loading through dynamic imports and route data loaders, keeping your initial bundle small.

Dynamic Component Imports

Route pages, layouts, and components can be dynamic imports. They are fetched only when the route is navigated to:

routes: {
  '/dashboard': {
    page: () => import('./pages/Dashboard.js'),
  },

  '/analytics': {
    layout: () => import('./layouts/AnalyticsLayout.js'),
    page: () => import('./pages/Analytics.js'),
  },
}

SafaRouter resolves dynamic imports automatically: if a page value is a string ending in .js or .mjs, it is loaded via dynamic import().

routes: {
  '/heavy-page': {
    page: '/pages/heavy.js',  // loaded on demand
  },
}

Route Data Loaders

Loaders fetch data asynchronously before rendering the page. They enable code splitting at the data level:

routes: {
  '/dashboard': {
    loader: async ({ params, query, router }) => {
      const response = await fetch('/api/dashboard')
      const data = await response.json()
      return data  // passed to page as {{ data.x }}
    },
    page: ({ data }) => `
      <h1>Dashboard</h1>
      <p>Users: ${data.users}</p>
      <p>Revenue: ${data.revenue}</p>
    `,
  },
}

Loading States

Show a loading indicator while the loader or page is being fetched:

routes: {
  '/slow': {
    loading: () => '<div class="spinner"><p>Loading…</p></div>',
    loader: async () => {
      await new Promise(r => setTimeout(r, 2000))
      return { data: 'loaded' }
    },
  },
}

SWR Loader Cache (v2.0.0)

SafaRouter's SWR (Stale-While-Revalidate) cache provides instant navigation for previously loaded routes:

const router = new SafaRouter({
  loaderCache: true,             // enable SWR (default)
  loaderStaleTime: 30000,        // 30s before cache is stale
  loaderCacheMaxSize: 100,       // max cached entries
})
Cache StateBehavior
Fresh (≤ staleTime)Returns cached data instantly — no fetch
Stale (> staleTime)Returns cached data + background re-fetch
No cacheAwaits fetch, then caches result

Concurrent requests to the same route are deduplicated — only one fetch is made.

Prefetching

Prefetch routes ahead of time for instant navigation:

// Manual prefetch
router.prefetch('/about')

// Auto-prefetch visible links
const router = new SafaRouter({
  prefetchStrategy: 'visible',  // uses IntersectionObserver
  pageDir: 'html-pages',
})

// Prefetch all links
const router = new SafaRouter({
  prefetchStrategy: 'all',
})

Route Caching

Loaded pages are cached in an LRU cache. Subsequent visits to the same route skip the fetch:

const router = new SafaRouter({
  cacheRoutes: true,     // default: true
  maxCacheSize: 50,      // max cached pages
})

// Clear cache when needed
router.clearCache()

Navigation Timeout

Set a timeout for navigation — useful for slow loaders:

const router = new SafaRouter({
  navigationTimeout: 10000,  // 10s timeout
})

Retry Failed Navigation

Retry a navigation with exponential backoff:

router.retry('/slow-api', {
  method: 'push',
  retries: 3,
})

Parallel Route Loaders (Slots)

Load multiple data sources in parallel using slots:

routes: {
  '/dashboard': {
    page: () => '<h1>Dashboard</h1>',
    slots: {
      sidebar: {
        loader: async () => fetch('/api/sidebar').then(r => r.json()),
      },
      analytics: {
        loader: async () => fetch('/api/analytics').then(r => r.json()),
      },
    },
  },
}
<aside data-safa-slot="sidebar"></aside>
<div data-safa-slot="analytics"></div>

Slot loaders run in parallel with the main loader via Promise.all.

Core Router for Minimal Bundle

For the smallest possible bundle, use the core router which excludes all optional features:

import { SafaRouter } from 'safa-router/core'

const router = new SafaRouter({
  target: '#app',
  routes: {
    '/': { page: () => '<h1>Home</h1>' },
    '/about': { page: () => '<h1>About</h1>' },
  },
})

router.start()

The core router is under 3 KB gzipped and includes route matching, layouts, middleware, history, and loaders — but not access control, maintenance mode, transitions, or smart components.
