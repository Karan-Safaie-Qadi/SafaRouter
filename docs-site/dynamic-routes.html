Dynamic Routes

SafaRouter supports three dynamic segment patterns for flexible URL matching.

Segment Patterns

Pattern Example URL params Description
[param] /blog/my-post { slug: 'my-post' } Matches one path segment
[...param] /docs/guide/intro { path: ['guide', 'intro'] } Matches one or more segments (catch-all)
[[...param]] /docs { path: [] } Matches zero or more segments (optional catch-all)

Basic Dynamic Segments [param]

routes: {
  '/blog': {
    children: {
      '[slug]': {
        page: ({ params }) => `
          <h1>Post: ${params.slug}</h1>
        `,
      },
    },
  },
}

With HTML-first routing, create a file named [slug].html:

<!-- html-pages/blog/[slug].html -->
<h1>Post: {{ params.slug }}</h1>

Catch-All Segments [...param]

Matches one or more path segments, returning an array:

routes: {
  '/docs': {
    children: {
      '[...path]': {
        page: ({ params }) => `
          <h1>Docs: ${params.path.join(' / ')}</h1>
        `,
      },
    },
  },
}

// /docs/guide/intro → params.path = ['guide', 'intro']
// /docs/api → params.path = ['api']

Optional Catch-All [[...param]]

Matches zero or more segments — useful for index pages that also accept deeper paths:

routes: {
  '/docs': {
    children: {
      '[[...path]]': {
        page: ({ params }) => `
          <h1>${params.path.length
            ? `Docs: ${params.path.join(' / ')}`
            : 'Docs Home'}</h1>
        `,
      },
    },
  },
}

// /docs → params.path = []
// /docs/guide → params.path = ['guide']

Accessing Params

Route parameters are available in multiple places:

// In a page component
({ params }) => `<h1>${params.slug}</h1>`

// In a loader
loader: async ({ params }) => {
  return await fetch(`/api/posts/${params.slug}`).then(r => r.json())
}

// In a guard
guard: ({ params }) => {
  return params.id ? true : '/'
}

// In template placeholders (HTML files)
// {{ params.slug }} → "my-post"
// {{ params.id }} → "42"

Type-Safe Parameters (TypeScript)

import { defineRoute, defineRoutes, SafeRouteParams } from 'safa-router/typed'

const blogRoute = defineRoute('/blog/[slug]')
// Type of blogRoute.params → { slug: string }

const docRoute = defineRoute('/docs/[...path]')
// Type of docRoute.params → { path: string[] }

type BlogParams = SafeRouteParams<'/blog/[slug]'>
// { slug: string }

Route Groups (group)

Route groups organize routes logically without affecting the URL:

routes: {
  '(main)': {
    layout: ({ children }) => `
      <header>Main Layout</header>
      <main>${children}</main>
    `,
    children: {
      'profile': {
        children: {
          '[id]': {
            page: ({ params }) => `
              <h1>Profile: ${params.id}</h1>
            `,
          },
        },
      },
      'settings': {},
    },
  },
}

// URL: /profile/123 — NOT /main/profile/123

Route groups are especially useful for:

  • Sharing layouts between sections without nesting in the URL
  • Organizing authentication routes (auth)/login, (auth)/register
  • Grouping routes by feature for large applications
💡

Order matters. Static routes take priority over dynamic routes. /blog/new matches before /blog/[slug].
